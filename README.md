# 🚀 105 проектов для FULLSTACK-разработчика и DevOps

## 📋 О проекте
Это мой личный путь от системного программирования до DevOps-инженера и архитектора.
Каждый проект — это отдельная практическая задача, которая приближает меня к пониманию того, как работают современные системы.

## 🗺 План прохождения

### **УРОВЕНЬ 0: Фундамент. C и Linux (Задачи 1-25)**
- **Блок 1: C — Язык, на котором всё построено (1-15)**
  - От Hello World до клиент-сервера на сокетах
- **Блок 2: Linux — Среда обитания (16-25)**
  - Bash-скриптинг, системные утилиты, автоматизация

### **УРОВЕНЬ 1: Скриптовый гигант — Python (Задачи 26-50)**
- **Блок 3: Python — Автоматизация (26-40)**
  - Системные задачи, работа с файлами, метрики
- **Блок 4: Python — Структуры данных и Алгоритмы (41-50)**
  - LRU Cache, BFS/Дейкстра, пулы потоков

### **УРОВЕНЬ 2: Веб-разработка и Базы данных (Задачи 51-75)**
- **Блок 5: FastAPI — Высокопроизводительные API (51-65)**
  - CRUD, JWT, WebSockets, Middleware
- **Блок 6: SQL — Властелин данных (66-75)**
  - JOIN, индексы, транзакции, оконные функции

### **УРОВЕНЬ 3: Контейнеризация и оркестрация (Задачи 76-85)**
- **Блок 7: Docker — Упаковка приложений (76-85)**
  - Dockerfile, Compose, сети, volumes, healthcheck

### **УРОВЕНЬ 4: Большая сборка (Задачи 86-100)**
- **Блок 8: Интеграция всего и вся (86-100)**
  - FastAPI + Celery + Redis + Nginx
  - CI/CD, мониторинг, Terraform, Ansible

### **БОНУСНЫЙ УРОВЕНЬ: Архитектор (Задачи 101-105)**
- **Блок 9: Проектирование систем (101-105)**
  - HighLoad архитектура
  - SLA/SLO/SLI
  - Disaster Recovery
  - Миграции без даунтайма
  - Architecture Decision Records

## 🔧 Как пользоваться
1. Каждая задача в отдельной папке с номером и названием
2. В папке обязательно есть README.md с описанием и статусом
3. После выполнения задачи обновляется прогресс
4. Код должен быть рабочим и задокументированным
